# BMAD 方法论使用记录

> 本项目使用 [BMAD-METHOD](https://github.com/bmad-code-org/BMAD-METHOD)（Brownfield Method Mapping）在 3 天内从想法到最小可行原型。

---

## 使用场景

课程设计实训（4 天周期），一人全栈开发。通过 BMAD 的头脑风暴 → 架构分析 → 里程碑规划流程，系统性地完成：

1. **头脑风暴**：从 PC 端单机应用升级为边缘-云协同架构
2. **技术深化**：三层架构的 36 个技术参数矩阵
3. **MVP 路径**：6 个里程碑 + 决策树 + 验收标准

---

## 约束识别

通过 Constraint Mapping 识别：
- i.MX6ULL 性能不是约束（实测验证 640x480 @ 25 FPS）
- 真正的约束是时间（一周）和人力（一人）
- 技术约束都有成熟的 Python 库可解决
