# 系统架构文档

**项目名称**：SAM-3 双范式工业视觉分析系统
**生成日期**：2026-01-03
**架构类型**：单体 Web 应用（Monolith）
**部署模式**：本地运行

---

## 1. 系统概述

### 1.1 项目定位
基于 VLM 语义引导与 SAM-3 开放词汇模型的工业视觉分析系统，支持三种分析范式：
- **范式 A**：VLM 引导的开放词汇实例分割
- **范式 B**：SAM-3 Purify + PaDiM 离线异常检测
- **范式 C**：VLM 缺陷框 → SAM 精分割

### 1.2 核心能力
- ✅ 零样本语义分割
- ✅ 工业异常检测
- ✅ 高精度缺陷定位
- ✅ GPU 加速推理
- ✅ 实时交互界面

## 3. 三范式架构

### 3.1 范式 A：VLM 引导的开放词汇实例分割

```
用户输入图像 + 关键词
    ↓
VLM 语义理解
    ↓
生成语义提示（Prompt）
    ↓
SAM-3 分割推理
    ↓
实例掩码输出
```

**特点**：
- 零样本能力（无需训练）
- 支持自然语言描述
- 实时交互

**适用场景**：
- 开放词汇物体分割
- 语义探索
- 快速原型验证

### 3.2 范式 B：SAM-3 Purify + PaDiM 离线异常检测

```
正常样本图像集
    ↓
SAM-3 背景净化
    ↓
ResNet 特征提取
    ↓
PaDiM 特征分布建模
    ↓
测试图像输入
    ↓
特征对比 + 异常定位
    ↓
异常热图输出
```

**特点**：
- 离线训练模式
- 无需异常样本
- 高精度异常定位

**适用场景**：
- 工业质检
- 缺陷检测
- 异常监控

### 3.3 范式 C：VLM 缺陷框 → SAM 精分割

```
用户输入图像
    ↓
VLM 缺陷检测（粗定位）
    ↓
生成边界框
    ↓
SAM-3 精细分割
    ↓
精确掩码输出 + 指标计算
```

**特点**：
- 两阶段检测
- 高精度分割
- 自动指标评估（IoU、精度、召回率）

**适用场景**：
- 精准缺陷分析
- 质量评估
- 自动化检测

## 4. 数据流设计

### 4.1 图像处理流程

```
原始图像（PIL/NumPy）
    ↓
预处理（resize, normalize）
    ↓
模型推理（SAM-3/VLM）
    ↓
后处理（掩码生成、边界框提取）
    ↓
可视化渲染
    ↓
Streamlit 展示
```

### 4.2 会话状态管理

Streamlit 使用 `session_state` 管理用户会话：

```python
session_state = {
    "models_ready": bool,          # 模型是否已加载
    "models_tuple": tuple,         # 模型实例缓存
    "selected_paradigm": str,      # 当前选择的范式
    "uploaded_image": PIL.Image,   # 上传的图像
    "inference_results": dict,     # 推理结果
    # ... 其他状态
}
```

## 5. 模型管理

### 5.1 模型加载策略

**懒加载机制**：
- 应用启动时不加载模型
- 用户点击"初始化模型"后才加载
- 加载后缓存到 `session_state`

**优势**：
- 加快首屏加载速度
- 避免不必要的资源占用
- 支持按需加载

### 5.2 模型列表

| 模型 | 来源 | 大小 | 用途 |
|------|------|------|------|
| SAM-3 | ModelScope | ~2GB | 实例分割 |
| ResNet-18/50 | PyTorch | ~100MB | 特征提取 |
| VLM | DashScope API | N/A | 语义理解 |

### 5.3 GPU/CPU 自适应

```python
DEVICE = "cuda" if torch.cuda.is_available() else "cpu"
```

- 自动检测 CUDA 可用性
- GPU 优先，CPU 回退
- 透明切换，无需用户干预

## 8. 性能特性

### 8.1 推理性能

| 模型 | GPU (RTX 4060) | CPU (i7) |
|------|----------------|----------|
| SAM-3 | ~200ms/图 | ~2s/图 |
| VLM API | ~500ms/图 | ~500ms/图 |
| PaDiM | ~100ms/图 | ~500ms/图 |

### 8.2 内存占用
- **GPU 显存**：~4GB（SAM-3 + ResNet）
- **系统内存**：~2GB（应用运行时）
- **模型缓存**：~3GB（磁盘）

### 8.3 并发能力
- **当前**：单用户单会话
- **限制**：Streamlit 单线程模型
- **扩展**：可部署多实例（Docker + 负载均衡）

## 10. 可扩展性设计

### 10.1 模块化架构
- **Core 与 UI 解耦**：算法层可独立测试和复用
- **范式独立**：每个范式独立实现，互不影响
- **组件化 UI**：通用组件可复用

### 10.2 扩展点

#### 新增范式
1. 在 `ui/` 下创建 `paradigm_d.py`
2. 在 `app_final.py` 中添加路由
3. 复用 `core/` 中的算法模块

#### 新增模型
1. 在 `core/models.py` 中添加加载逻辑
2. 在 `core/` 下创建推理模块
3. 在 UI 中调用新模型

#### 新增数据源
1. 在 `ui/components.py` 中添加输入组件
2. 在 `core/cv_utils.py` 中添加数据处理
3. 适配到现有范式

## 11. 技术债务与改进方向

### 11.1 当前限制
- ❌ 单机部署，无法多用户并发
- ❌ 静态图像输入，不支持视频流
- ❌ 依赖外部 API（VLM），网络不稳定时受影响
- ❌ 模型体积大，首次加载慢

### 11.2 改进方向
- ✅ **边缘-云协同**：支持分布式部署
- ✅ **视频流支持**：实时视频分析
- ✅ **模型优化**：量化、剪枝、蒸馏
- ✅ **缓存机制**：结果缓存、模型预热
- ✅ **监控告警**：性能监控、异常告警
